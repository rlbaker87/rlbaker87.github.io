AI Upskilling Program

A practice-first AI upskilling program designed for a healthcare L&D team operating under real governance constraints — built to run alongside production work, not instead of it.

10 Sessions Over 6 Months
70/20/10 Learning Model

The Challenge

The Learning Design and Technology team at a large regional health system was already using AI tools, but without shared standards, approved platforms, or any governance structure. Every team member reported using at least one unapproved tool. The team needed a program that could build consistent skills and responsible habits without pulling people away from active production work for extended training blocks.

The solution couldn't be a course. It had to be a system — one that built capability incrementally, created peer accountability, and produced infrastructure that would outlast the program itself.

Solution

This is a 10-session series designed around how the team actually works — in short windows, under real deadlines, with real tools.

70% Real Work Application Every session applies directly to tasks team members are already doing
20% Peer Learning Structured sharing, prompt swaps, and success story documentation
10% Formal Instruction Short, targeted input — frameworks, governance guidance, new tool orientation
  • Designed 10 fully self-contained 60-minute sessions running twice a month over six months. Self-contained structure means capacity constraints or schedule conflicts never permanently disqualify a participant — anyone can re-enter without being lost.
  • Built an AI Champions peer-leadership model with four internal Champions driving facilitation and sustaining adoption between sessions, eliminating dependence on any single facilitator or outside trainer and distributing ownership across the team.
  • Adapted governance frameworks from peer healthcare organizations into a Green/Yellow/Red decision tree mapped to the organization's existing ITS three-tier framework — rather than building compliance guidance from scratch or waiting on formal policy development.
  • Positioned governance as a relationship-building tool with ITS and Information Security, not a compliance exercise — treating IT partners as collaborators in the program rather than gatekeepers to route around.
  • Constructed shared infrastructure during the program itself: a SharePoint-based Prompt Library organized by ADDIE phase, a Success Stories database with a standard impact template, and an AI Verification Log for content governance. These are live tools built alongside the work, not end-of-program deliverables.
  • Applied a Kirkpatrick four-level evaluation plan with pulse checks after every session, ATD competency self-assessments at sessions 1, 5, and 10, and time-savings tracking tied to named recurring tasks the team performs every week.
Why This Approach

Most AI upskilling programs treat governance as an afterthought and build toward a future state the team can't sustain. This program builds the governance relationship and the shared infrastructure at the same time as the skills — so what's left at program close is a working system, not a slide deck.

Shared Infrastructure Built During the Program

These tools are live and in use from session one — not artifacts delivered at program close.

Prompt Library

SharePoint-based library organized by ADDIE phase. Team-contributed, tagged, and searchable. Target: 20+ entries by month 6.

Success Stories Database

Standardized impact template for documenting real AI wins. Feeds evaluation data and builds internal evidence for continued investment.

AI Verification Log

Content governance documentation for AI-assisted outputs. Supports audit readiness and establishes responsible use habits as standard practice.
